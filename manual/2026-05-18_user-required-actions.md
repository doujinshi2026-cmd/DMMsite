# ユーザー作業マニュアル 2026-05-18

## 目的

このプロジェクトを進めるために、ユーザー側で準備・判断が必要な項目を整理する。

## 1. アカウント・APIキーの準備

以下はユーザー側で取得・管理が必要。

- DMM Affiliate APIの利用に必要なID
- DMMアフィリエイトID
- X Developerアカウント
- X APIの投稿用キー・トークン
- GitHubリポジトリ
- Cloudflareアカウント
- Cloudflare Pages用プロジェクト

想定する環境変数名:

```env
DMM_API_ID=
DMM_AFFILIATE_ID=
X_API_KEY=
X_API_SECRET=
X_ACCESS_TOKEN=
X_ACCESS_SECRET=
```

注意:

- APIキーやアクセストークンをチャット、Markdown、Git管理対象ファイルに書かない。
- ローカルでは `.env`、GitHubでは GitHub Secrets、Cloudflareでは環境変数として登録する。

## 2. 規約・審査の確認

ユーザー側で最新の公式情報を確認する必要がある。

- DMM/FANZAアフィリエイトの利用規約
- DMM Affiliate APIの利用条件
- Xの自動投稿・成人向け投稿に関するルール
- 画像利用、試し読み画像、PR表記の扱い

確認結果によって、以下の実装方針が変わる可能性がある。

- 画像を直接保存するか、外部URL参照にするか
- 投稿文に必ず入れる表記
- 投稿頻度
- 成人向けコンテンツの表示制御

## 3. サイト方針として決めること

実装前に決めておくと迷いが減る項目。

- サイト名
- ドメイン名
- 主に扱うジャンル
- 扱わないジャンル
- レビュー文のトーン
- 感情タグの初期セット
- 1日あたりのX投稿数
- セール作品を優先するか、新着作品を優先するか

初期の感情タグ候補:

- 甘々
- 背徳感
- 切ない
- 緊張感
- イチャイチャ
- 距離感
- 続きが気になる

## 4. 次回までにユーザーが行うとよいこと

必須ではないが、次の実装段階に入る前に決めておくとよい。

- サイト名を1つ決める。
- 最初に扱うジャンルを3つまでに絞る。
- DMM/FANZA、X、Cloudflare、GitHubのアカウント準備状況を確認する。
- APIキーは取得しても、このリポジトリ内には保存しない。
